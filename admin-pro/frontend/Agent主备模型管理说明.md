# Agent 主/备用模型管理功能说明

## 🎯 功能概述

Agent 管理系统现在支持为每个 Agent 单独配置主模型和备用模型，提供更强的可用性保障。

## 🚀 功能特性

### 1. **主/备用模型配置** ✅
- **主模型**：Agent 优先使用的模型
- **备用模型**：当主模型不可用时自动切换的模型
- **智能切换**：支持自动/手动切换策略
- **跨提供商**：建议选择不同提供商的模型作为备用

### 2. **界面展示优化** ✅
- **直观显示**：Agent 列表清晰显示主/备用模型状态
- **配置状态**：一目了然地看到是否配置了备用模型
- **警告提示**：未配置备用模型时显示建议提示

### 3. **配置对话框** ✅
- **分区展示**：主模型和备用模型分别配置
- **智能验证**：防止主模型和备用模型相同
- **策略选择**：支持自动/手动切换模式
- **实时预览**：配置前显示即将使用的模型

## 📋 使用指南

### 配置主/备用模型：

1. **进入 Agent 管理页面**
   - 访问：`/agents`
   - 查看当前所有 Agent 的模型配置状态

2. **配置单个 Agent**
   - 点击「配置主/备用模型」按钮
   - 选择主模型（必选）
   - 选择是否启用备用模型
   - 如启用，选择备用模型（建议不同提供商）
   - 选择切换策略（推荐自动切换）

3. **保存配置**
   - 点击「保存配置」
   - 系统自动更新 `openclaw.json` 配置文件
   - 配置立即生效

### 最佳实践：

#### ✅ **推荐配置组合**
```
主模型: claudeagent/claude-3-5-haiku-latest
备用模型: minimax/MiniMax-M2.7
策略: 自动切换

主模型: openrouter/openai/gpt-4o  
备用模型: claudeagent/claude-sonnet-4-6
策略: 自动切换
```

#### ⚠️ **注意事项**
- **不同提供商**：主备模型选择不同提供商，避免单点故障
- **切换策略**：推荐「自动切换」，检测到主模型故障时立即切换
- **成本考虑**：备用模型可选择成本较低的选项
- **功能对等**：确保备用模型支持所需功能

## 🔧 技术实现

### 前端改进：
- ✅ Agent 列表显示主/备用模型状态
- ✅ 配置对话框支持主/备用模型设置
- ✅ 模型选择器防止重复选择
- ✅ 实时验证和预览功能

### 后端实现：
- ✅ `POST /api/agent/update-model` 支持主/备用模型参数
- ✅ `openclaw.json` 配置文件格式扩展：
  ```json
  {
    \"agents\": {
      \"list\": [{
        \"id\": \"hc-coding\",
        \"model\": \"claudeagent/claude-3-5-haiku-latest\",
        \"primaryModel\": \"claudeagent/claude-3-5-haiku-latest\",
        \"fallbackModel\": \"minimax/MiniMax-M2.7\",
        \"fallbackStrategy\": \"auto\"
      }]
    }
  }
  ```

### 兼容性：
- ✅ **向后兼容**：保留 `model` 字段，确保现有系统正常运行
- ✅ **渐进升级**：可逐步配置备用模型，不影响当前使用
